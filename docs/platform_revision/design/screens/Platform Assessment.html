01

What's working

Working well
Message components

Chat and messaging is the social backbone of the product. The components are clean, well-structured and feel native. Don't touch them.

Working well
Match Centre & Scores

The calendar and list views for fixtures are clear and functional. Good information hierarchy. The typography scale works at this density.

Working well
Bets & Predictions

The bet card pattern — deadline timer, reward pill, action button — is one of the clearest information layouts in the app. Solid work.

Working well
Pill token on the pitch

The horizontal nameplate with number badge is a big improvement over the old circle tokens. The concept is right; it just needs scaling up.

Working well
Type system

Archivo Black + JetBrains Mono is a strong, distinctive pairing. It's the identity. The problem is scale and application — not the fonts themselves.

Working well
Mobile structure

The mobile layout follows the same component system well. The squad pitch on mobile is a tighter, better-considered version of the desktop view.

02

What needs to change

01 /06 Critical

The dark is too dark

The app sits on #080A0E — essentially pure black. There's almost no surface variation between the page background, sidebar, cards, and hover states. Everything collapses into the same darkness.

  • Background (#080A0E), cards (#0F1218), hover (#161B25) differ by only 6–10 lightness units. Imperceptible on most screens.
  • Small JetBrains Mono labels at 10–11px on near-black require effort to read — users shouldn't be working hard at a glance.
  • On laptop screens in normal daylight conditions the interface reads as oppressive, not premium.
  • The Forza gaming aesthetic (pure black background) works for a game HUD. A daily-use web product needs more breathing room.

Fix: Raise all surface levels ~10 lightness units. Background → #0D1219. Cards → #131B26. Hover → #192233. The hierarchy becomes visible without changing the aesthetic.

02 /06 Critical

Navigation has three competing layers

When inside a league, the user is simultaneously presented with three navigation systems that each claim prime vertical space. The mental model breaks down completely.

Layer 1
Scores League Squad Live Market
Layer 2
▣ MANAGE SQUAD ▣ MARKET
Layer 3
Leaderboard Frontpage Bets Betting Auctions Chat Stats Admin

Layer 2 (MANAGE SQUAD / MARKET) isn't navigation — it's shortcuts. They belong in the sidebar or as contextual actions, not a persistent 48px strip.

  • "Bets" and "Betting" as separate tabs is confusing — users can't tell them apart without clicking both.
  • The "← Back" affordance from league → league list is tiny and easy to miss.
  • 8 tabs in the league hub is too many — it signals feature overload rather than a cohesive product.
  • The league name is truncated to "PREMIER FANT..." — a core identity element that shouldn't be cropped.

League Hub · Stats tab — three concurrent nav systems visible simultaneously

Fix: Remove the MANAGE SQUAD/MARKET sticky bar. Move them to the left sidebar as contextual actions when inside a league. Consolidate "Bets" and "Betting" into one tab. Consider grouping Auctions under Bets.

04 /06 Moderate

Typography is too small across the board

The type scale was tuned for a high-density broadcast display, not a web product used on laptop screens in all kinds of lighting. Almost every label needs to go up 1–2px.

  • JetBrains Mono at 10–11px is the system's workhorse but it's at the very edge of comfortable reading — any antialiasing variation makes it feel smaller.
  • The eyebrow text ("TACTICAL SHEET", "COMPETITIVE CENTER") is barely visible at its current size and contrast.
  • The BUY buttons in the player market are too small to be confident primary actions.
  • Club codes below player names on the pitch read fine when zoomed but fail the glance test at normal viewing distance.
  • The sidebar nav labels (SCORES, SQUAD, LEAGUE…) with icons beside them are correct in concept but the text needs to be 13px, not 11px.

Fix: Eyebrow labels → 11px minimum. Player name on pitch → 14px. Sidebar nav → 13px. Metadata below names → 10px (not 9px). Button labels → 11px. These are 1–2px changes with outsized readability impact.

05 /06 Moderate

The Forza brand is a liability

The "FORZA | FANTASY LEAGUE" wordmark and the surrounding gaming aesthetic were the right call for a pitch deck. They're not the right call for a product that needs to feel warm, social, and accessible.

  • The Forza name implies a formal partnership or licensing agreement. If that isn't in place, the product is exposed.
  • The "broadcast data overlay" aesthetic (mono everything, pure black, cyan only) works for a game UI that players interact with for hours. A fantasy app is checked multiple times a day in spare moments — it needs to feel lighter and faster.
  • The gaming aesthetic is at odds with the social features (chat, bets, frontpage) that are the product's real differentiators.
  • "ALPHA V0.1 · FORZA ECOSYSTEM" in the sidebar footer undermines confidence for pilot users.

Fix: Rename to FantasyKit (or a clean standalone name) and create an independent visual identity. Preserve the typography system — it's strong. Change the branding, not the bones.

The FORZAKIT wordmark — implying a partnership that may not be sustainable

League select screen — cold, gamified aesthetic on a social feature

06 /06 Minor

League select has no context

The league picker is the first contextual screen users hit after login. It's a plain list with names and nothing else — no current rank, no live status, no unread activity indicator.

  • "TEST_2_H2H_DRAFT" and "TEXT_WC_CONTROL_CLASSIC" are test/internal names leaking into the pilot — these undermine trust and readability.
  • No visual distinction between active leagues, completed leagues, and draft-pending leagues.
  • No rank or points shown — the one number that would make users want to tap immediately.
  • The screen feels like a database admin panel, not the gateway to a social competition.

Fix: Show rank, points and unread count on each league card. Add a status badge (LIVE / DRAFT / COMPLETE). This is a quick data addition with a large perceived quality improvement.

Recommendation

Ship A now.
Explore B or C for next season.

Direction A is not a compromise — it's the right move for a live pilot with real users. The core platform has everything it needs to succeed. The dark palette, the type system, and the token concept are all worth keeping. What needs to go is the friction: the oppressive blackness, the three-layer nav, the inconsistent screens. These are solvable in two weeks without touching the data model or the component tree.

In parallel, run one design sprint exploring Direction B or C for the next season launch. The question isn't "fix the app" — it's "what does FantasyKit feel like when it's fully itself, not a Forza skin?"
