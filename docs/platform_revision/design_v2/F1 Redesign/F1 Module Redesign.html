Frontrow — F1 module design proposal

Kit Light · reconciled tokens · aligned to Clubhouse Core (V2_DESIGN) and the Tennis module

Reused as-is

All global chrome — 220px dark sidebar, competition top-bar pill strip, card/elevation language, type system (Archivo Black / Archivo / JetBrains Mono), 6px radius — is lifted directly from Clubhouse Core, same as Tennis. F1's own secondary nav differs from Tennis's 2-item pattern: F1 needs five destinations (Calendar/Picks/Standings/Report/Season), so it keeps its own tab row rather than borrowing Tennis's Home/Leaderboard toggle.

Where red actually goes

Today red only lives in nav chrome and Standings. This pass gives F1 a real identity across all six screens, but deliberately narrow: --f1 marks brand and action — the Home header shell, the next-race countdown banner, the active round pill, podium P1 slot, Save/Submit buttons, Report's total figures and podium-viz top block, the Standings toggle. It is never used for correctness signaling — right/wrong picks in Report and Season Bets stay on the app's existing positive/danger tokens, same as everywhere else in the product. That split matters because F1's own brand red (#E10600, Ferrari-saturated) sits close enough to the semantic danger red (#B91C1C, a duller brick) that blurring the two would make a correct-pick chip look like an error state. Keeping them separate is the load-bearing decision in this brief.

The checkered flag as the one motif

Rather than invent new iconography, a thin checkered strip — the sport's single most recognizable object — recurs as a quiet accent: under the next-race banner corner, at the top of the Report's podium visualization. Small, monochrome, never decorative filler.

Home: countdown first, calendar second

The next-race card is solid red, full width, with the countdown numerals treated at display scale — the one thing on the page that outranks the 24-row calendar below it. The calendar itself borrows Tennis's dimming trick (upcoming rows at 42% opacity) but swaps status pills for F1's own vocabulary: qualifying/live/finished-with-winner-name.

Race Picks: one flexible slot, not three components

Podium, DNF and Team are stable card shapes every round. Special Category is not — it's built as a single component that swaps its inner render (driver picker / team picker / toggle buttons) rather than three separate screens, so the round-to-round variability lives in one place in the form, not scattered across conditional layouts.

Report: the podium block earns its place

The old aspirational mockup's podium visualization is real craft worth keeping — it's added here above the pick-by-pick comparison table, using actual race results (not scores) so it reads as "what happened" before the table explains "how you did against it."

Bets are global, not per-paddock

The paddock switcher in the F1 header changes whose leaderboard you see — it does not imply a different set of picks. Nothing in the Race Picks or Season Bets screens is paddock-scoped UI; only Standings and the Report's comparison-to-others (out of scope here) are.

The dark-shell header is the one deliberate break from Tennis's cream header — it reads as a cockpit instrument, not a season ledger, and gives the checkered strip somewhere natural to sit. Next-race banner is solid F1 red, full-width, with the countdown at display type scale — it's the only thing on the page that outranks the calendar. Season list reuses Tennis's dimming trick for upcoming rows but swaps in F1 status language (picks open/qualifying/live) and shows the winner's name once a race is scored, not just a result tag. Mobile bottom nav carries all five F1 destinations including Season (as CLUB→ replaced by dedicated slots per the brief's nav gap) — Season Bets is reached via the Paddocks tab's card grid on mobile, matching current shipped behavior; we did not add a sixth nav slot since five is already the practical max.
