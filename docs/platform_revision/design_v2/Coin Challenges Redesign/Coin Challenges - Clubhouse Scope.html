00Composition point of view

Coin Challenges — Clubhouse scope + Freeform bets

Kit Light · reconciled tokens · desktop 1440 & mobile 390, paired by screen

What's changing, what isn't

Additive to the existing, already-Kit-Light screen: a real member picker replaces the raw-ID field, a Competitor / Freeform choice opens the create flow, and a net-new declare → confirm/dispute → owner arbitrates lifecycle exists for Freeform only (Competitor bets still auto-resolve off the GW total). Layout, wallet sidebar and card anatomy are untouched.

Bet-type identity — the color call

Competitor keeps gold — it already owns stakes, the primary CTA and the incoming-card border. Freeform takes accent blue — unclaimed on this screen for bet identity, and already reads as "the other kind of thing" against gold elsewhere in the product.

The dispute state needed its own visual language

Disputed/pending-arbitration is a lifecycle status, not a bet type — it gets a neutral dashed treatment instead of gold or blue. The owner's decision action borrows purple, already in the system as the GK position tone, repurposed here as a one-off "owner/arbitration" role marker.

Member picker & mobile progressive disclosure

The opponent list reuses the Members tab row (avatar, name, search). On mobile, bet-type, opponent, bet detail and stake become four separate full screens instead of one long sheet.

How this doc is organized

Each numbered screen (S01–S10) pairs its desktop artboard directly with the matching mobile 390 artboard, side by side, for easy tracking.
