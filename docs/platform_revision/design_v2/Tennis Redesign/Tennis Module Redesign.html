Frontrow — Tennis module design proposal

Kit Light · reconciled tokens · aligned to the Clubhouse Core system (V2_DESIGN)

Reused as-is

All chrome — 220px dark sidebar, competition top-bar pill strip, card/elevation language, type system (Archivo Black / Archivo / JetBrains Mono), 6px radius — is lifted directly from the Clubhouse Core handoff. Nothing about global chrome changes for Tennis; only the in-competition secondary nav is new (Home · Leaderboard, two items, matching the sparse 3-item mobile bottom nav from the brief).

The green question

The brief asked whether Tennis should lean hard into --ten the way F1 leans into red, or stay a quieter "clean scoreboard." We went quieter: green is reserved for primary actions and identity moments — the season accent bar, the "pick your squad" state, pick buttons, the roster progress fill — never for dense data or decoration. Captain and urgency states use gold, not green, so the QF Captain window reads as its own distinct moment rather than "more tennis." This matches the module's own rhythm: checkpoint-based and calm, not a live dashboard.

The calendar problem

14 tournaments in one list risks reading as a spreadsheet. We solved it with three moves: a surface-colour dot per row (hard/clay/grass/indoor) for instant scanning, a single status pill per row whose colour does all the "what do I do now" work (green = act now, gold = urgent, accent = in progress, outline = done, muted = not yet), and upcoming rows dimmed to 42% opacity and non-interactive so the eye lands on what's actionable. The action banner above the calendar is the one un-missable element on the page — solid green fill, white text — and it's the only thing on Home that disappears entirely when nothing needs the user's attention.

The roster picker

Built as four independent tier blocks (never one long list) so the 1-2-2-2 slot structure is legible at a glance, with a sticky roster-summary rail on the right that fills in as picks are made — the same pattern a checkout flow uses to keep the user oriented across a long form. A player already used elsewhere greys out immediately. The Ace Card sits below the tiers, not above — it's a bonus decision, not the entry point.

QF Captain — urgent, not alarming

Gold, not red or danger. A visible countdown chip communicates urgency without borrowing the danger-red vocabulary reserved for destructive actions elsewhere in the product.

ATP Finals

The named, escalating scoring tiers (Unforced Error → The Perfect Slate) are treated as a standing reference card, always visible while picking — the closest thing this module has to a slot-machine payout ladder, and worth the showcase treatment the brief invited.
