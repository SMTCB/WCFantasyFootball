## Task 2 — BUG-13: Admin panel functions use session JWT

**Problem:** `callFunction` in `AdminSeedScreen.jsx` always uses `ANON_KEY` (`sb_publishable_*` format). Functions with `verify_jwt=true` reject this key. Admin panel buttons (Sync Fixtures, Sync Players, Discover Tournament, Resolve Bets) all fail with auth errors.

**Fix:** Get session JWT via `supabase.auth.getSession()`, fall back to anon key only when no session exists.

**Files:**
- Modify: `src/screens/AdminSeedScreen.jsx` lines 22–36

- [ ] **Step 1: Update the import and `callFunction` helper**

  Find this block in `AdminSeedScreen.jsx` (lines 22–36):
  ```js

    ? `${import.meta.env.VITE_SUPABASE_URL}/functions/v1`
    : 'https://sssmvihxtqtohisghjet.supabase.co/functions/v1';

  const ANON_KEY = import.meta.env.VITE_SUPABASE_ANON_KEY || 'sb_publishable_IQF1vJEiydutRmDa6XgDUA_FHTlWX0b';

  async function callFunction(name, body) {
    const res = await fetch(`${FUNCTIONS_URL}/${name}`, {
      method:  'POST',
      headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${ANON_KEY}` },
      body:    JSON.stringify(body),
    });
    return res.json();
  }
  ```

  Replace with:
  ```js
  import { supabase, FUNCTIONS_BASE } from '../lib/supabase';

  const FUNCTIONS_URL = FUNCTIONS_BASE
    ?? 'https://sssmvihxtqtohisghjet.supabase.co/functions/v1';

  const ANON_KEY = import.meta.env.VITE_SUPABASE_ANON_KEY
    ?? 'sb_publishable_IQF1vJEiydutRmDa6XgDUA_FHTlWX0b';

      body:    JSON.stringify(body),

  }
  ```

  Note: `supabase` was already imported on line 19 — the new import replaces it with a named import that also pulls `FUNCTIONS_BASE`.

## Task 3 — BUG-14: Transfer buy/sell use raw fetch with session JWT

**Problem:** `supabase.functions.invoke()` with `sb_publishable_*` key does not correctly surface the response body for the `buy` action. The function is called, returns HTTP 200, but `data.players` and `data.budget_remaining` are lost, so the squad display never updates. `sell` appears to work because the calling code in `MarketScreen` observes the takenMap optimistic update regardless of returned data.

**Fix:** Replace both `buy` and `sell` with raw `fetch()` calls using the session JWT (same pattern as the working `callFunction` in `AdminSeedScreen`).

**Files:**
- Modify: `src/hooks/useTransfer.js`

- [ ] **Step 1: Add the import and local helper**

  At the top of `useTransfer.js`, update the import line:
  ```js
  import { supabase, FUNCTIONS_BASE } from '../lib/supabase';
  ```

  Then add a module-level helper just below the imports:
  ```js
  async function invokeTransfer(body) {
    if (!FUNCTIONS_BASE) return { ok: false, error: 'Supabase not configured' };
    const { data: { session } } = await supabase.auth.getSession();
    const token = session?.access_token ?? import.meta.env.VITE_SUPABASE_ANON_KEY;
    const res = await fetch(`${FUNCTIONS_BASE}/process-transfer`, {
      method:  'POST',
      headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${token}` },
      body:    JSON.stringify(body),
    });
    if (!res.ok) {
      let msg = 'Transfer failed';
      try { const b = await res.json(); msg = b?.error ?? msg; } catch { /* ignore */ }
      return { ok: false, error: msg };
    }
    return res.json();
  }
  ```

- [ ] **Step 2: Replace `buy` body**

  Find the `buy` useCallback (lines ~55–90 in `useTransfer.js`). Replace the entire inner body with:


## Task 5 — BUG-12: LiveScreen shows correct tournament fixture on first render

**Problem:** `fetchAll` reads `activeLeague?.tournamentId` at call time. On the very first call (mount), `activeLeague` is `null` (initial state). The next-fixture query has no tournament filter → returns the nearest fixture in the DB (often a WC fixture). Leagues are loaded later in the same call but `activeLeague` state only updates on the next render cycle.

**Fix:** After loading memberships within `fetchAll`, if `activeTournamentId` was null on entry, re-run the next-fixture query with the now-resolved tournament ID.

**Files:**
- Modify: `src/screens/LiveScreen.jsx`

- [ ] **Step 1: Find the next-fixture query block (lines ~356–368)**

  It currently looks like:
  ```js
  // Fetch next upcoming fixture — filtered by active league's tournament (BUG-12 fix)
  if (!activeFixIds.length) {
    let upcomingQ = supabase.from('fixtures')
      .select('id, home_team, away_team, status, kickoff_at, tournament_id')
      .eq('status', 'scheduled')
      .gt('kickoff_at', new Date().toISOString())
      .order('kickoff_at', { ascending: true })
      .limit(1);
    if (activeTournamentId) upcomingQ = upcomingQ.eq('tournament_id', activeTournamentId);
    const { data: upcomingData = [] } = await upcomingQ;
    if (upcomingData?.length) setNextFixture(upcomingData[0]);
  } else {
    setNextFixture(null);
  }
  ```

  No change needed here — this is correct as-is.

- [ ] **Step 2: Add a re-fetch after enrichedLeagues is built (around line ~485)**

  Find the block that ends with:
  ```js
  if (!initialSet.current && enrichedLeagues.length) {
    setActiveLeague(enrichedLeagues[0]);
    initialSet.current = true;
  }
  ```

  Add immediately after it:
  ```js
  // BUG-12 fix: on first load activeTournamentId was null because activeLeague wasn't set yet.
  // Now that we know the user's leagues, correct the next-fixture if it needs a tournament filter.
  const resolvedTId = activeTournamentId ?? enrichedLeagues[0]?.tournamentId ?? null;
  if (!activeFixIds.length && !activeTournamentId && resolvedTId) {
    const { data: corrected = [] } = await supabase
      .from('fixtures')
      .select('id, home_team, away_team, status, kickoff_at, tournament_id')
      .eq('status', 'scheduled')
      .gt('kickoff_at', new Date().toISOString())
      .eq('tournament_id', resolvedTId)
      .order('kickoff_at', { ascending: true })
      .limit(1);
    if (corrected?.length) setNextFixture(corrected[0]);
  }
  ```
