# Draft + H2H Mode — System Design

**Complete specification for the head-to-head parallel competition layer.**

---

## Overview

Draft + H2H is a third league mode that runs a head-to-head competition in parallel with the standard cumulative fantasy points table. Every matchday, each manager is paired against one opponent. The outcome (win/draw/loss) awards H2H points. At the end of the tournament, two independent leaderboards exist: classic fantasy points and H2H points.

This mode is an opt-in layer on top of the existing draft format. It is new-leagues-only and fully feature-flagged — disabling it on a league has zero effect on fantasy points or any other system.

---

## Core Mechanics

### H2H Scoring

Win/draw/loss point values are **configurable per league** via `league_config` JSONB:

```json
{
  "h2h_win_pts": 5,
  "h2h_draw_pts": 2,
  "h2h_loss_pts": 0
}
```

Defaults (`5/2/0`) are used if keys are absent. Changing these values requires only a JSON update — no code change, no migration.

### Round Resolution Gate

H2H results are only computed when **every fixture in the matchday is finished** (`status = 'finished'`). This uses the same `roundComplete` check as auto-sub logic in `calculate-scores`.

For multi-day matchdays (e.g. WC R32 spread over 5 days), H2H resolves only after the final whistle of the last match in that round. The scoring cron will tick through the matchday without writing H2H results until that condition is met.

Resolution is idempotent: if H2H rows already exist for a matchday (scores populated), the step skips.

### Bye Rule

When a league has an odd number of managers, one manager receives a bye each matchday. Bye = **automatic win (5 pts, or the configured `h2h_win_pts` value)**. The bye rotates through all managers evenly as part of the generated round-robin schedule.

### Round-Robin Schedule

Pairings follow a standard round-robin:
- Everyone plays everyone once, then the cycle repeats
- Schedule is pre-generated by the admin and stored in `h2h_schedule`
- Late joiners are handled by generating the schedule after the final manager count is confirmed
- The schedule can start from **any matchday** — if the competition is already underway, H2H begins from the next unplayed matchday; past matchdays are not retroactively scheduled

## Database Schema

### `leagues` table change

```sql
ALTER TABLE leagues ADD COLUMN h2h_enabled boolean DEFAULT false;
```

### `h2h_schedule` table

Stores both the schedule (before play) and results (after resolution). Scores and H2H points are null until the matchday resolves.

```sql
CREATE TABLE h2h_schedule (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  league_id       uuid NOT NULL REFERENCES leagues(id) ON DELETE CASCADE,
  matchday_id     text NOT NULL,           -- e.g. '426-r4'
  home_user_id    uuid REFERENCES auth.users(id),  -- null if bye
  away_user_id    uuid REFERENCES auth.users(id),  -- null if bye
  is_bye          boolean NOT NULL DEFAULT false,
  bye_user_id     uuid REFERENCES auth.users(id),  -- set when is_bye=true
  home_score      numeric,                 -- fantasy points, populated post-match
  away_score      numeric,
  home_h2h_pts    integer,                 -- 5/2/0, populated post-match
  away_h2h_pts    integer,
  resolved_at     timestamptz,
  UNIQUE (league_id, matchday_id, home_user_id, away_user_id)
);
```

### H2H Standings (view)

```sql
CREATE VIEW h2h_standings AS
SELECT
  league_id,
  user_id,
  COUNT(*) FILTER (WHERE h2h_pts = win_pts) AS wins,
  COUNT(*) FILTER (WHERE h2h_pts = draw_pts) AS draws,
  COUNT(*) FILTER (WHERE h2h_pts = 0)        AS losses,
  SUM(h2h_pts) AS total_h2h_pts
FROM (
  -- home rows
  SELECT league_id, home_user_id AS user_id, home_h2h_pts AS h2h_pts FROM h2h_schedule WHERE resolved_at IS NOT NULL
  UNION ALL
  -- away rows
  SELECT league_id, away_user_id AS user_id, away_h2h_pts AS h2h_pts FROM h2h_schedule WHERE resolved_at IS NOT NULL AND is_bye = false
  UNION ALL
  -- bye rows
  SELECT league_id, bye_user_id AS user_id, bye_h2h_pts AS h2h_pts FROM h2h_schedule WHERE resolved_at IS NOT NULL AND is_bye = true
) t
GROUP BY league_id, user_id;
```

### RLS Policies

- `h2h_schedule` SELECT: league members (`is_league_member`)
- `h2h_schedule` INSERT/UPDATE: service-role only (admin RPC + scoring hook are SECURITY DEFINER)

## Admin RPC — `generate_h2h_schedule`

```sql
generate_h2h_schedule(p_league_id uuid, p_start_matchday_id text)
```

- Commissioner-only (auth check inside function)
- Fetches all league members at time of call
- Generates a round-robin sequence covering all remaining matchdays from `p_start_matchday_id`
- Handles odd manager count: inserts bye rows with `is_bye = true, bye_user_id = <manager>`
- Idempotent for a given starting matchday — calling again after generation is a no-op (or can regenerate if not yet resolved)
- Does NOT retroactively schedule matchdays that already have resolved `fantasy_points` rows

## Scoring Hook — `calculate-scores` integration

After the standard fantasy points calculation for a matchday, `calculate-scores` checks:

1. Is `h2h_enabled = true` for this league?
2. Are all fixtures for this `matchday_id` in `status = 'finished'`? (`roundComplete`)
3. Does `h2h_schedule` have unresolved rows for this `matchday_id`?

If all three: fetch each pairing's fantasy points totals, compute win/draw/loss using `league_config` values, update `h2h_schedule` rows with scores + H2H points + `resolved_at`, then write a gazette entry.

### Gazette Entry

One `activity` gazette entry per league after resolution:

```
⚔️ Matchday 3 H2H Results
Alice 85 pts beat Bob 72 pts       +5
Charlie 91 pts beat Dave 68 pts    +5
Emma 77 pts — BYE                  +5
```

## Frontend

### League Creation Wizard

Three mode cards (third is new):

| Card | `league_mode` | `h2h_enabled` |
|---|---|---|
| Classic | `classic` | `false` |
| Draft | `draft` | `false` |
| Draft + H2H | `draft` | `true` |

### Mode Tag

Pill badge next to league name in header and league selector:
- `CLASSIC` — neutral
- `DRAFT` — current style
- `DRAFT · H2H` — gold accent

### H2H Tab (League Screen)

Visible only when `h2h_enabled = true`. Two sub-sections:

**Standings** — H2H table: Rank / Manager / W–D–L / H2H Pts

**Schedule** — All matchdays, grouped by round. Each fixture shows manager names, scores (or "vs TBD"), and outcome badge (W/D/L) once resolved.

**Empty state** (calendar not yet generated by admin): "Calendar not yet available — check back later."

### Leaderboard

When `h2h_enabled = true`:
- Primary sort: H2H Pts (default)
- Columns: Rank / Manager / H2H Pts / W–D–L / Fantasy Pts
- Fantasy Pts column retained as secondary reference

### Admin Tab — H2H Calendar Section

New section alongside the Draft section:

- **Before generation**: matchday picker (defaults to next unplayed matchday) + "Generate Calendar" button
- **After generation**: read-only fixture list grouped by matchday
- Late-start supported: admin picks any future matchday as the start point

---

## Safety & Revert

- All H2H logic is gated behind `h2h_enabled = false` (default). Toggle off → all H2H UI disappears, no data loss, no effect on fantasy points.
- New tables only — zero changes to `fantasy_points`, `squads`, `leagues` (beyond the one new boolean), or any existing scoring logic.
- H2H resolution failure logs an error but does not block fantasy points writes.
- Per-league opt-in — existing leagues see nothing new.
